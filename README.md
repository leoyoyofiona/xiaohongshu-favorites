# 小红书收藏导航

[English](./README.en.md) | [日本語](./README.ja.md)

把小红书收藏夹从“堆着不看”变成“能找、能分、能复看”的 macOS 桌面工具。

## 痛点

- 收藏越来越多，真正要用的时候翻不到。
- 小红书收藏夹没有适合长期整理和复盘的桌面视图。
- 论文、工具、教育、灵感混在一起，后续很难二次利用。
- 看到一篇内容想导出原文和图片，手动保存很麻烦。

## 这个 App 能做什么

- 同步 Chrome 里已经登录的小红书收藏夹。
- 自动去重、自动分类，把内容归到主分类。
- 左侧分类浏览，中间列表查看，右侧阅读原文内容。
- 支持标为重点、标记已读、上一篇/下一篇快速浏览。
- 支持一键下载当前文章的原文和全部图片。
- 支持导出 `.dmg` 安装包，方便分发给朋友。

## 当前工作方式

- 同步阶段依赖 `Chrome`。
- 请先在 Chrome 打开你的小红书收藏夹页面，再回 App 点击同步。
- 这是当前最稳的方案，比内嵌网页同步更不容易触发平台风控。

## 安装

### 方式 1：直接安装 `.dmg`

1. 打开 `小红书收藏导航.dmg`
2. 将 `小红书收藏导航.app` 拖到 `Applications`
3. 第一次打开如果被 macOS 拦截，右键 `打开` 一次，或者去“系统设置 > 隐私与安全性”点允许

### 方式 2：从源码运行

要求：

- macOS 15+
- Xcode Command Line Tools
- Swift 6.2

运行：

```bash
swift run XHSOrganizerApp
```

## 使用方法

### 1. 同步收藏夹

1. 在 Chrome 打开你的小红书收藏夹
2. 打开 App
3. 点击 `同步小红书`
4. 点击 `从当前 Chrome 收藏夹同步`

### 2. 浏览内容

- 左侧：分类导航
- 中间：收藏列表
- 右侧：原文内容、图片、分类调整、重点标记

### 3. 下载当前文章

在右侧详情区域点击 `下载原文`：

- 会导出 `原文.txt`
- 会导出 `原文链接.txt`
- 会把当前文章图片一起下载
- 文件默认保存到 `下载/小红书收藏导出/`

## 项目结构

- `Sources/XHSOrganizerApp`：macOS SwiftUI 界面与桌面端逻辑
- `Sources/XHSOrganizerCore`：数据模型、同步导入、分类、搜索
- `scripts/build_dmg.sh`：打包 `.app` 和 `.dmg`
- `scripts/generate_app_icon.py`：生成 App 图标

## 打包

生成安装包：

```bash
./scripts/build_dmg.sh
```

输出：

- `dist/小红书收藏导航.app`
- `dist/小红书收藏导航.dmg`

## 已知限制

- 当前同步依赖 Chrome，而不是 Safari 或 App 内嵌浏览器。
- 小红书部分文章在网页端有访问限制，原文抓取成功率取决于链接完整度和页面可见性。
- 旧数据如果是早期同步得到的裸链接，可能需要重新同步一次，才能提升原文显示比例。

## 适合谁

- 收藏了很多论文、工具、教程、教育内容的重度小红书用户
- 想把碎片收藏变成长期可用资料库的人
- 想在桌面端更高效整理小红书收藏的人
